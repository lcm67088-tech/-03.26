## 공통 인프라 설계

### LLM 비용 부담 방식
- **플랫폼 부담** (유저 허들 제거, 남용은 플랜 제한으로 통제)
- 운영 비용 근거: 1,000명 × 10회/월 × Gemini Flash 기준 ≈ **4,000원/월** (무시 가능 수준)
- 고급 기능(GPT-4o, Claude Sonnet)은 **Pro/Enterprise 플랜 전용**으로 차별화

### 권장 모델 매핑 (2026년 3월 기준)

| 기능 | 기본 모델 (Free/Starter) | 고급 모델 (Pro+) |
|---|---|---|
| 소개글 개선 제안 | Gemini 1.5 Flash | GPT-4o / Claude 3.5 Sonnet |
| 리뷰 감성 분석 | Gemini 1.5 Flash | GPT-4o mini |
| 월간 성과 리포트 | Gemini 1.5 Flash | GPT-4o |
| 완성도 점수 | Gemini 1.5 Flash | — (기본으로 충분) |
| 리뷰 답변 초안 | Gemini 1.5 Flash | GPT-4o / Claude 3.5 Sonnet |

### 플랜별 월 사용 제한 (안)

| 플랜 | 소개글 | 감성분석 | 리포트 | 완성도 | 답변초안 |
|---|---|---|---|---|---|
| Free | 1회 | 1회 | — | 무제한 | 3회 |
| Starter | 5회 | 5회 | 1회 | 무제한 | 10회 |
| Pro | 20회 | 무제한 | 3회 | 무제한 | 무제한 |
| Enterprise | 무제한 | 무제한 | 무제한 | 무제한 | 무제한 |

> **완성도 점수**: 룰 기반 계산이 메인, LLM은 개선 제안 코멘트에만 사용 → 사실상 무제한

### LLM 없는 모드 처리

## 기능 1. 소개글 개선 제안

### 목적
현재 소개글에서 SEO 관점의 누락 키워드를 감지하고, 개선된 소개글 초안을 자동 생성한다.

### 입력 컨텍스트
```
- 업체명, 카테고리
- 현재 소개글 (p.introduction)
- 대표키워드 목록 (p.keywords)
- 순위키 목록 (isRankKw: true)
- 메뉴 목록 (p.menus[].name)
- 영수증 리뷰 태그 top10
```

### LLM 요청 구조
```
역할: 네이버 플레이스 SEO 전문가
작업:
  1. 현재 소개글에서 누락된 키워드 목록 추출
  2. 키워드가 자연스럽게 포함된 개선 소개글 초안 작성 (200~400자)
  3. 개선 포인트 3가지 bullet 형태로 설명

제약:
  - 과장 표현 금지
  - 키워드 나열식 문장 금지 (자연스러운 문장 필수)
  - 원래 소개글 톤앤매너 유지
```

### 반환 구조
```json
{
  "missing_keywords": ["치즈카츠", "혼밥", "주차"],
  "improved_intro": "역삼동 한가운데 위치한 ...",
  "improvement_points": [
    "메뉴명 '치즈카츠'가 소개글에 없어 검색 노출 약함",
    "혼밥 관련 언급 추가 시 관련 키워드 유입 가능",
    "주차 가능 여부 명시 권장"
  ]
}
```

### UI 표시
- **접근 위치**: 플레이스 상세 → 인사이트 탭 또는 새 'AI 제안' 섹션
- 현재 소개글과 개선 소개글 **나란히 비교 표시 (diff 하이라이트)**
- 누락 키워드 배지 표시
- [소개글에 적용] 버튼 → 직접 수정 모달로 연결 (자동 저장 아님)

### 제한
- 입력 소개글 없으면 → "소개글이 없습니다. 직접 입력 후 분석하세요" 안내

## 기능 2. 리뷰 감성 분석

### 목적
영수증 리뷰 + 블로그·카페 리뷰 전체를 분석해 긍정/부정/중립 비율과 주요 키워드를 추출한다.

### 입력 컨텍스트
```
- 영수증 리뷰: [{ stars, text, tags, item }] (최근 50개)
- 블로그/카페 리뷰: [{ title, content }] (최근 20개)
```

### 처리 방식
- **1차 룰 기반**: 별점 기준 분류 (4~5 = 긍정, 3 = 중립, 1~2 = 부정)
- **2차 LLM**: 텍스트 내용 기반 감성 재분류 + 키워드 클러스터링

### LLM 요청 구조
```
역할: 고객 리뷰 분석 전문가
작업:
  1. 리뷰 텍스트 전체를 긍정/부정/중립으로 분류
  2. 주요 긍정 키워드 top10 추출 (예: 맛있다, 친절, 가성비)
  3. 주요 부정 키워드 top5 추출 (예: 대기 길다, 주차 불편)
  4. 감성 요약 1~2 문장
```

### 반환 구조
```json
{
  "positive_ratio": 72,
  "neutral_ratio": 18,
  "negative_ratio": 10,
  "positive_keywords": ["맛있다", "친절", "가성비", "재방문", "청결"],
  "negative_keywords": ["대기 길다", "주차 불편", "양 적다"],
  "summary": "전반적으로 음식 맛과 서비스에 만족도가 높으나, 대기 시간과 주차 관련 불만이 반복 언급됩니다."
}
```

### UI 표시
- **접근 위치**: 플레이스 상세 → 리뷰 탭 상단 or 인사이트 탭
- 도넛 차트: 긍정/중립/부정 비율
- 긍정 키워드: 초록 배지, 부정 키워드: 빨간 배지
- 감성 요약 텍스트 한 줄 표시
- 분석 기준: "최근 50개 영수증 리뷰 + 20개 블로그 리뷰 기준"

### 제한
- 리뷰 0개일 때 → "리뷰가 없어 분석할 수 없습니다" 표시
- 텍스트 없는 별점 리뷰만 있을 때 → 룰 기반 비율만 표시

## 기능 3. 월간 성과 리포트 자동 생성

### 목적
한 달간의 순위 변동·리뷰 증감·크롤링 데이터를 종합해 읽기 쉬운 성과 리포트를 자동 생성한다.

### 입력 컨텍스트
```
- 기간: 직전 30일
- 순위 데이터: 키워드별 일자별 rank (rank history)
- 리뷰 증감: 이번달 vs 지난달 영수증/블로그 리뷰 수
- 크롤링 로그: session 이력
- 플레이스 기본 정보 (이름, 카테고리)
```

### LLM 요청 구조
```
역할: 디지털 마케팅 성과 분석 전문가
작업:
  1. 순위 변동 요약 (상승/하락/유지 키워드별)
  2. 리뷰 증감 평가
  3. 이달의 성과 핵심 3가지
  4. 다음 달 집중 권장 액션 3가지
  5. 전체 성과 1~2 문장 요약

출력 형식: 카드형 리포트 (한글, 비전문가도 이해 가능)
```

### 반환 구조
```json
{
  "period": "2026년 2월",
  "headline": "순위 키워드 2개 TOP5 진입! 리뷰 증가세 지속",
  "rank_summary": {
    "improved": ["강남 돈카츠 맛집 (8위→3위)", "역삼 맛집 (12위→7위)"],
    "declined": ["강남 맛집 (4위→9위)"],
    "stable": ["강남 돈카츠 (5위 유지)"]
  },
  "review_summary": "영수증 리뷰 +14건, 블로그 리뷰 +3건 증가",
  "highlights": [
    "작업키 '강남 돈카츠 맛집' 3위 달성으로 목표 키워드 순위 상승 기여",
    "영수증 리뷰 꾸준한 증가세 (월 평균 대비 +18%)",
    "블로그 리뷰 품질 지수 개선됨"
  ],
  "actions": [
    "하락 중인 '강남 맛집' 작업키 재점검 필요",
    "다음 달 블로그 리뷰 2건 이상 추가 권장",
    "메뉴 탭 사진 보강으로 클릭률 개선 시도"
  ],
  "overall": "이번 달은 대체로 좋은 성과를 기록했습니다. 특히 맛집 계열 키워드 상승이 두드러집니다."
}
```

### UI 표시
- **접근 위치**: 인사이트 탭 → '월간 리포트' 섹션
- 카드형 레이아웃: 헤드라인 → 순위 요약 → 리뷰 요약 → 하이라이트 → 액션
- [리포트 복사] / [PDF 내보내기] 버튼 (추후 구현)
- 생성 날짜 표시 (매월 1일 자동 생성 or 수동 [리포트 생성] 버튼)

### 생성 트리거
- **자동**: 매월 1일 00:00 백그라운드 태스크
- **수동**: [이번 달 리포트 생성] 버튼 (Pro 이상만 수동 재생성)

## 기능 5. 플레이스 정보 완성도 점수

### 목적
플레이스의 SEO·노출 관점에서 정보 완성도를 0~100점으로 수치화하고, 개선 체크리스트를 제공한다.

### 채점 항목 (룰 기반, 100점 만점)

| 항목 | 배점 | 조건 |
|---|---|---|
| 소개글 | 20점 | 없음 0점 / 50자 미만 10점 / 50자 이상 20점 |
| 사진 수 | 15점 | 0개 0점 / 1~4개 8점 / 5개+ 15점 |
| 메뉴 등록 수 | 15점 | 0개 0점 / 1~2개 8점 / 3개+ 15점 |
| 영업시간 | 10점 | 미등록 0점 / 등록 10점 |
| 순위키 등록 | 15점 | 0개 0점 / 1개 8점 / 2~3개 15점 |
| 작업키 등록 | 10점 | 0개 0점 / 1~2개 5점 / 3개+ 10점 |
| 전화번호 | 5점 | 미등록 0점 / 등록 5점 |
| 영수증 리뷰 수 | 10점 | 0개 0점 / 1~9개 5점 / 10개+ 10점 |

> **총 100점**, LLM은 개선 코멘트 텍스트 생성에만 활용 (룰 기반이 메인)

### LLM 활용 (코멘트 생성)
- 낮은 점수 항목 2~3개를 컨텍스트로 전달
- LLM이 업체 특성에 맞는 구체적 개선 제안 문장 생성
- 예: "소개글에 대표 메뉴 '치즈카츠'를 언급하면 관련 검색 유입이 늘어날 수 있습니다"

### 반환 구조
```json
{
  "total_score": 67,
  "grade": "B",
  "items": [
    { "name": "소개글", "score": 10, "max": 20, "status": "warn", "comment": "50자 미만 — 대표 메뉴 언급 권장" },
    { "name": "사진", "score": 15, "max": 15, "status": "ok", "comment": null },
    { "name": "메뉴", "score": 15, "max": 15, "status": "ok", "comment": null },
    ...
  ],
  "top_improvements": [
    "소개글을 보강하면 +10점 예상",
    "순위키를 3개 모두 등록하면 +7점 예상"
  ]
}
```

### UI 표시
- **접근 위치**: 플레이스 카드 상단 or 인사이트 탭
- 원형 게이지 + 점수 숫자 (큰 폰트)
- 등급 표시: S(90+) / A(80+) / B(70+) / C(60+) / D(60 미만)
- 항목별 체크리스트 (✅ ok / ⚠️ warn / ❌ fail)
- 개선 시 예상 점수 상승 표시

## 기능 6. 리뷰 답변 초안 생성

### 목적
부정 리뷰 또는 특정 리뷰에 대해 브랜드 톤에 맞는 답변 초안을 자동 생성한다.

### 입력 컨텍스트
```
- 리뷰 텍스트, 별점, 태그, 아이템명
- 업체명, 카테고리
- (선택) 업체 톤앤매너 설정 (공손형 / 친근형 / 비즈니스형)
```

### LLM 요청 구조
```
역할: 자영업 고객 서비스 전문가
작업:
  - 별점과 리뷰 내용 기반으로 적절한 답변 초안 생성
  - 긍정 리뷰: 감사 인사 + 재방문 유도
  - 부정 리뷰: 사과 + 개선 의지 + 재방문 유도 (방어적 표현 금지)
  - 중립 리뷰: 공감 + 보완 포인트 언급

제약:
  - 2~4 문장 (너무 길면 읽지 않음)
  - 업체명 언급 포함
  - 개인 정보 추측 금지
  - 과도한 마케팅 문구 금지
```

### 반환 구조
```json
{
  "draft": "안녕하세요, 돈카츠 가게입니다. 불편하신 점이 있으셨군요, 정말 죄송합니다. 대기 시간 단축을 위해 운영 방식을 개선 중이니, 다음에 방문 시 더 나은 경험을 드릴 수 있도록 최선을 다하겠습니다. 다시 한번 찾아주시면 감사하겠습니다 🙏",
  "tone": "공손형",
  "type": "negative"
}
```

### UI 표시
- **접근 위치**: 리뷰 카드 우측 하단 [답변 초안] 버튼
- 부정 리뷰(별점 1~2)에 버튼 자동 강조 표시
- 클릭 시 초안 텍스트 생성 → 인라인 편집 가능한 텍스트영역으로 표시
- [복사] 버튼으로 클립보드 복사 → 유저가 네이버 앱에서 직접 게시
- 톤 선택: 공손형 / 친근형 (라디오 버튼)

### 주의
- **실제 답변 게시 자동화는 불가** (네이버 API 미제공)
- 초안 제공 → 유저가 직접 복사 후 네이버 앱에서 게시하는 방식

| 월간 성과 리포트 | 인사이트 탭 상단 (가장 눈에 띄는 위치) |
| 완성도 점수 | 플레이스 카드 오른쪽 or 인사이트 탭 첫 번째 카드 |
| 리뷰 답변 초안 | 리뷰 카드 → [답변 초안] 버튼 |

---

## 구현 우선순위 및 단계

### Phase 1 (MVP): 기능 5 + 6
- 완성도 점수: 룰 기반 계산 → LLM 코멘트 없이도 즉시 구현 가능
- 리뷰 답변 초안: 단순 LLM 1-shot 호출, 구현 난이도 가장 낮음
- **예상 소요**: 1~2일

### Phase 2: 기능 2
- 리뷰 감성 분석: 기존 리뷰 데이터 활용, 배치 처리 고려
- **예상 소요**: 2~3일

### Phase 3: 기능 1
- 소개글 개선 제안: 컨텍스트 구성 복잡, 프롬프트 튜닝 필요
- **예상 소요**: 2~3일

### Phase 4: 기능 3
- 월간 리포트: 데이터 집계 파이프라인 + 자동 트리거 구현 필요
- **예상 소요**: 3~5일
